| 时间 | 任务 | 交付物 | 状态 |
|------|------|--------|------|

| 2-4h | 写 `ai-client.py`，接入通义千问 API（DeepSeek 降级）| AI 能生成题目 | ⬜ |

| 时间 | 任务 | 交付物 | 状态 |

| 时间 | 任务 | 交付物 | 状态 |

| 22-24h | 协助前端/UI 修最后 bug | 无 block 问题 | ⬜ |
| 24-26h | 辅助演示人员做最后彩排 | demo 流畅 | ⬜ |

### 给你的技术决策

- **V1 前端直调 AI API**：你把 API Key 给前端，他直接测，不等你后端
- **V2 后端代理 AI API**：前端改 `api-client.js` 的 `BASE_URL`，指向你的 `localhost:5000`
- **降级优先级**：AI 挂 → 本地题库 → 游戏继续，不能崩

## 👤 成员 B：前端开发

### 当前阶段（0-6h）：V1 单文件可玩版本

| 时间 | 任务 | 交付物 | 状态 |
|------|------|--------|------|
| 0-1h | 读 `api-contract.json`，确认理解接口格式 | 群里提问（如有） | ⬜ |
| 1-3h | 写 `index.html`（单文件，含 CSS + JS），实现 Canvas 渲染 + 触控事件 | 打开浏览器能玩 | ⬜ |
| 3-5h | 实现 `QuestionProvider`（先读本地 JSON）+ `Judge` + `ScoreManager` | 能判对错 + 记分 | ⬜ |
| 5-6h | 接 `Renderer`，把题目、按钮、分数画到 Canvas 上 | 视觉基本完整 | ⬜ |

### 中期阶段（6-18h）：V2 接后端 + 特效

| 时间 | 任务 | 交付物 | 状态 |
|------|------|--------|------|
| 6-9h | 写 `api-client.js`，封装 4 个 API 调用 | 前端能调后端 | ⬜ |
| 9-12h | 把 `QuestionProvider` 改成优先调后端 AI，降级到本地 | AI 题目能用 | ⬜ |
| 12-15h | 实现 `EffectManager`：按钮特效 + 音效 + 连击动画 | 游戏有"感觉" | ⬜ |
| 15-18h | 实现结算画面：成绩单 + 雷达图 + 分享按钮 | 一局能完整跑完 | ⬜ |

### 后期阶段（18-26h）：打磨 + 修 bug

| 时间 | 任务 | 交付物 | 状态 |
|------|------|--------|------|
| 18-22h | 抖音小游戏容器适配测试，修布局问题 | 容器内正常显示 | ⬜ |
| 22-24h | 跟后端联调挑战模式，修 bug | 挑战功能完整 | ⬜ |
| 24-26h | 协助演示彩排 | demo 流畅 | ⬜ |

### 给前端的注意事项

- **V1 不做后端调用**：先让游戏能跑，题目从 `questions-fallback.json` 读
- **Canvas 尺寸固定 375×667**：不要做响应式，抖音容器会处理缩放
- **`gameState` 全局对象**：所有状态存在这里，跟后端无关
